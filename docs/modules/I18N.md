## 1. Overview

Internationalization defines how the ATS supports multiple languages and locale-aware formatting. This covers the `next-intl` setup, translation workflow, locale-aware date/number/currency formatting, RTL support, and the scope of language coverage for MVP and post-MVP.

**Scope:**
- In scope: next-intl configuration, translation file structure, locale-aware formatting, RTL layout support, career page translations, email template localization, language detection strategy.
- Out of scope: Machine translation (manual translations only), per-field translations for user content (job descriptions are single-language), multi-language job postings (post-MVP).

export const locales = ['en-US', 'en-GB', 'es', 'fr', 'de', 'pt-BR', 'ar', 'ja'] as const;

## 4. Locale-Aware Formatting

### 4.1 Date Formatting

```typescript
// lib/i18n/formatters.ts
import { format } from 'date-fns';
import { enUS, enGB, es, fr, de, ptBR, ar, ja } from 'date-fns/locale';

const localeMap: Record<Locale, DateFnsLocale> = {
  'en-US': enUS, 'en-GB': enGB, es, fr, de, 'pt-BR': ptBR, ar, ja,
};

export function formatDate(date: Date, locale: Locale, style: 'short' | 'long' = 'short') {
  const patterns = {
    short: { 'en-US': 'MM/dd/yyyy', 'en-GB': 'dd/MM/yyyy', default: 'dd/MM/yyyy' },
    long: { 'en-US': 'MMMM d, yyyy', 'en-GB': 'd MMMM yyyy', default: 'd MMMM yyyy' },
  };

  return format(date, patterns[style][locale] ?? patterns[style].default, {
    locale: localeMap[locale],
  });
}
```

### 4.2 Currency Formatting

```typescript
export function formatCurrency(amount: number, currency: string, locale: Locale) {
  return new Intl.NumberFormat(locale, {
    style: 'currency',
    currency,
    minimumFractionDigits: 0,
    maximumFractionDigits: 0,
  }).format(amount);
}

// formatCurrency(75000, 'USD', 'en-US') → "$75,000"
// formatCurrency(75000, 'EUR', 'de')     → "75.000 €"
```

### 4.3 Relative Time

```typescript
export function formatRelativeTime(date: Date, locale: Locale) {
  const rtf = new Intl.RelativeTimeFormat(locale, { numeric: 'auto' });
  const diff = Date.now() - date.getTime();
  const minutes = Math.floor(diff / 60000);

  if (minutes < 60) return rtf.format(-minutes, 'minute');
  if (minutes < 1440) return rtf.format(-Math.floor(minutes / 60), 'hour');
  return rtf.format(-Math.floor(minutes / 1440), 'day');
}
```

## 5. RTL Support

### 5.1 Layout

```typescript
// app/layout.tsx
export default function RootLayout({ children, params }: { children: React.ReactNode; params: { locale: Locale } }) {
  const dir = ['ar'].includes(params.locale) ? 'rtl' : 'ltr';

  return (
    <html lang={params.locale} dir={dir}>
      <body>{children}</body>
    </html>
  );
}
```

### 5.2 CSS

```css
/* Tailwind CSS logical properties for RTL */
/* Use ms-* (margin-start) instead of ml-* (margin-left) */
/* Use ps-* (padding-start) instead of pl-* (padding-left) */

[dir="rtl"] .sidebar { /* Sidebar flips to right side */ }
```

### 5.3 RTL Components

shadcn/ui components support RTL via CSS logical properties. No component code changes needed — only CSS direction matters.

## 6. Career Page Localization

### 6.1 Language Detection

Career pages detect language from:
1. URL parameter: `/careers/{slug}?lang=es`
2. Browser `Accept-Language` header
3. Organization default locale

### 6.2 Localized Content

| Content | Translatable | Method |
|---------|-------------|--------|
| UI strings (buttons, labels) | ✅ | next-intl translation files |
| Job title/description | ❌ (MVP) | Single-language, written by recruiter |
| Application form labels | ✅ | Translation files |
| Email templates | ✅ | Per-locale email template variants |
| GDPR consent text | ✅ | Per-jurisdiction text in translation files |

## 7. Email Localization

### 7.1 Template Variables (D08 Compatibility)

Email templates use Handlebars `{{variable.path}}` syntax (D08 §3.3). Localized strings are injected as template variables:

```typescript
// When sending candidate email, resolve locale-specific strings
const locale = candidateLocale ?? orgDefaultLocale ?? 'en-US';
const t = await getTranslations(locale, 'emails');

const variables = {
  candidate: { name: candidate.full_name },
  job: { title: job.title },
  organization: { name: org.name },
  // Localized strings injected as variables
  strings: {
    greeting: t('greeting', { name: candidate.full_name }),
    cta_button: t('view_application'),
    footer: t('footer_text'),
  },
};
```

### 7.2 Candidate Locale Detection

Candidate locale is determined by:
1. Browser language at application time (stored in `applications.metadata.locale`)
2. Organization default locale (fallback)

## 9. Security Considerations

- **Translation injection:** Translation strings are escaped by next-intl before rendering. No raw HTML in translation values (use `rich` text API for formatting).
- **Locale validation:** Locale parameter validated against allowed list. Invalid locales fall back to `en-US`.
- **No user-generated translations:** Organizations cannot provide custom translations (prevents XSS via translation files). All translations are maintained by the development team.
